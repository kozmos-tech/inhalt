# Copy & tone

Rules for product copy - README, the site, dashboard UI text, and docs.

- Plain language, no jargon. Explain things so a non-expert understands.
- Monochrome / professional aesthetic. No decorative color, no childish styling.
- **No git / version-control framing.** Don't describe features in terms of
  commits, repos, branches, or history. Use "drafts", "save", "publish" instead.
- No em dashes in product copy.
- The product name is always capitalized: **Inhalt**.

## Positioning (one-liners to stay consistent with)

- Inhalt is an open-source, MCP-native CMS - the content layer is exposed through
  the Model Context Protocol, not a traditional admin UI or REST/GraphQL SDK.
- "The CMS your AI tools can run."
- The protocol is the API. No SDK, no glue code.
